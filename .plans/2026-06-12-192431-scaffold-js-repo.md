# Scaffold a clean JS repo

**Date:** 2026-06-12 19:24
**Task:** Create a clean, minimal JavaScript repository in text-dither.

## Goal
A fresh git repo with a minimal ESM package: package.json, stub module, passing test via node's built-in test runner, .gitignore, README. Committed on main.

## Input Information
- Directory is empty (greenfield).
- Node v26.0.0 installed - built-in test runner available, no dev dependencies needed.
- User asked for "a clean js repo" - no features, no tooling beyond the basics.

## Approach

### Step-by-step plan
1. `git init`, rename branch to main (done).
2. Write `package.json` - ESM ("type": "module"), `npm test` runs `node --test`, no dependencies.
3. Write `src/index.js` - stub export so the package has an entry point.
4. Write `test/index.test.js` - one test using node:test, proves the test pipeline works.
5. Write `.gitignore` - node_modules, dist, .env, .DS_Store.
6. Write `README.md` - name, usage, test command.
7. Run `npm test` to verify.
8. Write worklog, stage everything, commit.

### Key decisions
- **No dependencies, no linter/formatter:** user asked for clean and minimal; node:test covers testing with zero installs.
  - Alternatives considered: vitest/jest (adds node_modules for no current need), eslint+prettier (not requested).
- **ESM over CJS:** current default for new Node projects.

## Risks & Edge Cases
- None significant; greenfield scaffold.

## Files to Modify
- `package.json` - new, package manifest
- `src/index.js` - new, stub entry point
- `test/index.test.js` - new, smoke test
- `.gitignore` - new
