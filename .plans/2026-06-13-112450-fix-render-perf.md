# Fix UI freezing while adjusting settings

**Date:** 2026-06-13 11:24
**Task:** App freezes while dragging sliders. Find the render bottleneck and fix it at the root.

## Goal
Smooth slider interaction (no main-thread saturation) across the cellPx range, including small cells (many glyphs) and plot-preview mode.

## Input Information
- Slider `input` fires rapidly; each calls state.set -> rerender -> renderer.render, coalesced through one rAF, so at most one draw per frame. The freeze is therefore per-frame cost, not event flooding.
- draw() recomputes luminanceGrid (drawImage + getImageData + per-pixel loop) every render even when only tone params change and the grid is identical.
- Glyph loop sets `ctx.font` per glyph (continuous weight+size = a unique string each time) and uses save/translate/rotate/restore for rotated glyphs. Glyph count explodes at small cellPx (cellPx 7 on a ~960px canvas ~= 13k glyphs).
- Plot preview rebuilds Hershey polylines (passes + hatch) per glyph - far heavier than fillText.

## Approach

### Step-by-step plan
1. Instrument draw() (lum / layout / draw timings + glyph count), build, stress in browser at low cellPx and with preview on; read console; identify dominant phase. REMOVE instrumentation before commit.
2. Based on measurement, apply the high-impact fixes:
   - Cache the luminance grid keyed by (source, width, height, cols, rows); only resample when those change. Tone/text/warp/etc sliders then skip the resample entirely.
   - If the glyph draw loop dominates (expected): set `ctx.font` only when the string changes from the previous glyph, by quantizing the font string (round fontSize, keep integer weight). Glyphs are same-color ink and effectively non-overlapping, so optionally group by font string to set it once per group; globalAlpha stays per-glyph (cheap).
3. Keep behavior identical (same pixels within rounding); verify unit + e2e suites still green, including the preview round-trip e2e (toggle must restore the exact hash - any quantization must be deterministic).
4. Re-measure to confirm the freeze is gone; screenshot to confirm unchanged output.
5. Worklog, commit, push, confirm CI + deploy.

### Key decisions
- **Measure before fixing:** do not guess the bottleneck; instrument and read real timings.
- **Cache luminance, not layout:** layout is cheap-ish and depends on most params; the grid resample (GPU readback) is the wasteful repeat.

## Risks & Edge Cases
- Quantizing fontSize must stay deterministic or the preview-toggle e2e (exact hash restore) breaks.
- Luminance cache must invalidate on new image (setSource) and on cols/rows/size change.
- Grouping by font reorders draws; safe only because marks are same-color ink (order-independent compositing). Confirm visually.

## Files to Modify
- src/render/canvas.ts - luminance cache, draw loop
- possibly src/core/sampler.ts - expose a cache or cache in renderer
