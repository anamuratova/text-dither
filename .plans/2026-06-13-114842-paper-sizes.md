# Paper sizes (A1-A4 + custom) with image fill

**Date:** 2026-06-13 11:48
**Task:** Add paper size presets A1/A2/A3/A4 + custom, and make the image fill the chosen paper.

## Goal
A paper-size control drives both the on-screen page and the plot SVG export. The page has the paper's fixed aspect ratio; the image cover-fills it (scale to cover, crop overflow, no distortion).

## Input Information
- Today the screen canvas is sized to the IMAGE aspect; the plot page width is a single pageWidthMm slider (100-841) and the page height is derived from the image aspect. So the page is image-shaped, not paper-shaped.
- sampler.coverRect(cols, rows, width, height, sw, sh) already cover-fits the image into the canvas physical rect; once the canvas is paper-shaped, the image cover-fills the paper automatically (crops overflow).
- A-series portrait dims (mm): A4 210x297, A3 297x420, A2 420x594, A1 594x841.
- e2e only references #btn-plot-svg, #plot-stats, #chk-plot-preview; replacing #plot-width is safe.

## Approach

### Step-by-step plan
1. state.ts: PaperSize 'A4'|'A3'|'A2'|'A1'|'custom', Orientation 'portrait'|'landscape'. PlotSettings: drop pageWidthMm, add paperSize, orientation, customWidthMm, customHeightMm; keep marginMm. paperDimensions(settings) -> {widthMm, heightMm} (presets swap for landscape; custom uses literal dims). Default A4 portrait.
2. plotSvg.ts: rename printableWidthMm -> printableMm (axis-agnostic). PlotOptions: add heightMm. Page written as widthMm x heightMm (explicit, not derived). pxPerMm = canvasW / printableMm(widthMm, margin).
3. canvas.ts: import printableMm. PageGeometry {widthMm, heightMm, marginMm}. render(params, page, previewProfile|null). draw(): printable aspect = printableMm(h)/printableMm(w); fit the canvas inside its container (stage-wrap content box) preserving paper aspect, set explicit style width/height; image cover-fills via existing coverRect. drawPlotPreview uses pxPerMm = width / printableMm(w).
4. main.ts: build PageGeometry from paperDimensions + margin; pass to render and to toPlotSVG (widthMm, heightMm).
5. controls.ts: replace page-width input with paper-size select + orientation select + custom width/height number inputs + margin input. Disable orientation + custom inputs appropriately by selection.
6. styles.css: let JS drive canvas style width/height; keep centering, background, shadow.
7. Tests: plot.spec.ts (add heightMm to OPTS/literals, rename import, assert page width+height attrs = paper dims); new tests/state.spec.ts for paperDimensions (presets, landscape swap, custom literal); e2e: select A3 -> canvas aspect ~ A3 portrait (0.707) and Save plot SVG page = 297x420.
8. Verify typecheck/unit/e2e/build; screenshot A4 portrait + A3 landscape to confirm image fills + fits viewport. Worklog, commit, push, CI.

### Key decisions
- **Fill = cover-fit (crop), not fit (letterbox) or stretch:** "fill it" means no whitespace; stretch would reintroduce the aspect distortion just fixed. coverRect already does cover.
- **Canvas = printable area (paper minus margins):** what is on screen is exactly what plots; the plot maps the canvas into the printable rect and adds the margin border, so the SVG page equals the paper size exactly.
- **Orientation included:** paper sizes without orientation are not usable for landscape images; it is a 1-line dimension swap and integral to "paper sizes."
- **Fit canvas to container both axes:** portrait papers would otherwise overflow vertically; compute displayW = min(availW, availH/paperAspect) from the container content box (robust, avoids reading the canvas's own clientWidth circularly).

## Risks & Edge Cases
- Deterministic rendering must hold so the preview-toggle exact-hash e2e passes: container size is stable within a test, so canvas dims are stable.
- Custom dims of 0 / margin >= half page: printableMm floors at 1mm (already), so no divide-by-zero; clamp custom inputs to a sane min.
- Default changes the demo look (square rose cover-fills portrait A4 -> sides cropped). Expected per "fill it."
