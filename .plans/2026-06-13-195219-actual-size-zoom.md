# 100% (actual size) zoom toggle

**Date:** 2026-06-13 19:52
**Task:** Add a "100% zoom" option so the page is shown at real physical size, to check letter sizes before plotting.

## Goal
A Fit <-> 100% toggle. At 100%, the printable page area is displayed at actual mm (1 mm on paper = 1 CSS mm on screen, browser's 96px/inch reference) so the user can measure letter sizes against a ruler. The plot itself is unchanged - only the on-screen presentation scales.

## Input Information
- canvas.ts draw() derives the LAYOUT basis displayW from the HOST container (availW/availH), not from the canvas's own size, then sets canvas.style.width/height to fit. cols = floor(displayW/(0.62*cellPx)) -> the plot depends on the host width, not the canvas style size.
- So I can change canvas.style size for zoom WITHOUT changing the layout/plot, as long as displayW (host-derived) is computed the same way.
- Canvas represents the printable area (paper minus margins); the image fills it. Showing printable area at actual mm shows the letters at real plot size.
- stage-wrap is overflow:auto, display:flex centered - a canvas larger than the container (likely at 100% / large papers) must stay fully scrollable.

## Approach

### Step-by-step plan
1. state.ts: ZoomMode 'fit'|'actual'; add zoom to PlotSettings (default 'fit').
2. canvas.ts: render(params, page, previewProfile, zoom). draw() keeps layout/internal buffer from host-fit displayW (unchanged); only the canvas style size depends on zoom:
   - fit: style px = displayW/displayH (current)
   - actual: style = `${printW}mm` x `${printH}mm` (browser converts mm->px at 96/inch; no magic constant in code)
3. main.ts: pass state.getPlot().zoom to render.
4. controls.ts: checkbox "100% (actual size)" in the Plotter section toggling zoom.
5. styles.css: stage-wrap align/justify use `safe center` so an overflowing (100%) canvas scrolls fully instead of clipping at the top/left.
6. e2e: toggling 100% sets style.width/height to the printable mm (A4 portrait -> 190mm x 277mm) and leaves the internal buffer (c.width) unchanged (plot identical); unchecking restores px.
7. Verify typecheck/unit/e2e/build; screenshot at 100%. Worklog, commit, push, CI.

### Key decisions
- **Scale CSS only, keep layout from host width:** guarantees 100% shows the same plot at real size rather than re-tiling a different cell count. The pre-existing window-width dependence of the layout is untouched (separate concern).
- **Set style in mm, not a computed px constant:** the browser owns the mm->px reference (96/inch); accurate-as-possible without knowing the monitor, and the e2e can assert the exact 'mm' string.
- **Show the printable area at actual size** (not the full sheet): letters live in the printable area, so this is the size that matters for legibility; we do not render the margin border anyway.

## Risks & Edge Cases
- Real monitors are not exactly 96 CSS px/inch, so "actual size" is approximate (note to user).
- A huge canvas (A1 at 100%) overflows; `safe center` keeps it scrollable. Internal buffer stays host-fit sized (capped 1600), so very large papers look slightly soft at 100% - acceptable for a size check.
- Scrollbar appearing at 100% could shave a few px off host width on the next render (tiny cols shift). Negligible / absent with overlay scrollbars; noted.
