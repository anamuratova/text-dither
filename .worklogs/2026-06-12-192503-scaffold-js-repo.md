## Summary
Initialized git repo (branch main) and scaffolded a minimal zero-dependency ESM JavaScript package with a passing smoke test.

## Context & Problem
Empty directory text-dither. User asked for "a clean js repo" - nothing more specified, so the goal was the smallest correct starting point that builds and tests.

## Decisions Made

### Zero dependencies, node:test for testing
- **Chose:** Node's built-in test runner (`node --test`), no node_modules at all.
- **Why:** Node v26 is installed; built-in runner needs no install and keeps the repo clean.
- **Alternatives considered:**
  - vitest/jest - rejected, adds dependencies for no current need
  - eslint/prettier - rejected, not requested; can be added when there is real code to lint

### ESM module type
- **Chose:** "type": "module" in package.json.
- **Why:** Current default for new Node projects; avoids a CJS-to-ESM migration later.
- **Alternatives considered:**
  - CJS - rejected, legacy format

## Architectural Notes
Layout: `src/` for code, `test/` for tests, entry point `src/index.js` with a stub export. The stub exists only to prove the import/test pipeline; replace it when real functionality lands.
