# Fix CI typecheck failure: missing @types/node

**Date:** 2026-06-12 20:40
**Scope:** package.json, package-lock.json, tsconfig.json

## Summary
First CI run on GitHub failed in tsc with "Cannot find name 'Buffer'/'process'" (playwright.config.ts, tests/e2e/app.spec.ts) while the same command passed locally. Added @types/node as a devDependency and "node" to tsconfig "types".

## Context & Problem
After pushing commit c698be1 and enabling Pages, both CI and Deploy workflows failed at tsc --noEmit. Locally the identical command had passed during verification.

## Decisions Made

### Declare @types/node explicitly instead of relying on resolution
- **Chose:** npm i -D @types/node; "types": ["vite/client", "node"].
- **Why:** Root cause - a stray /Users/anna/Projects/node_modules/@types/node (parent directory of the repo) satisfied tsc's upward node_modules walk locally, masking the missing dependency. CI has no parent node_modules, so the build failed there. The dependency is real (Buffer/process in Playwright config and e2e tests), so it must be declared.
- **Alternatives considered:**
  - Separate tsconfig for e2e/node files only - rejected, extra config surface for no practical gain in a repo this size.
  - Removing Buffer/process usage from tests - rejected, they are the natural API for setInputFiles buffers.

## Architectural Notes
Lesson recorded: local typecheck green is not proof when a parent node_modules exists; CI is the arbiter.
