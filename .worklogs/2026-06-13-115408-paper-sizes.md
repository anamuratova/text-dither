# Paper sizes (A1-A4 + custom) with image fill

**Date:** 2026-06-13 11:54
**Scope:** src/state.ts, src/plot/plotSvg.ts, src/render/canvas.ts, src/main.ts, src/ui/controls.ts, src/styles.css, tests/plot.spec.ts, tests/state.spec.ts (new), tests/e2e/app.spec.ts

## Summary
Added A4/A3/A2/A1 + custom paper sizes with portrait/landscape orientation. The page now takes the paper's aspect ratio (both on screen and in the plot SVG export) and the image cover-fills it. Replaced the old single page-width slider.

## Context & Problem
Previously the screen canvas was image-shaped and the plot page width was a lone slider with the height derived from the image aspect - so the page was image-shaped, not a real paper. The user wanted standard paper sizes and the image to fill the chosen paper.

## Decisions Made

### Fill = cover-fit (crop), not fit or stretch
- **Chose:** the image scales to cover the paper, cropping overflow, no distortion.
- **Why:** "fill it" means no whitespace; stretching would reintroduce the aspect distortion fixed earlier. The sampler's coverRect already does physical-space cover, so once the canvas is paper-shaped the image cover-fills automatically.

### Canvas = printable area (paper minus margins)
- **Chose:** the canvas (and the glyph field) takes the printable aspect; the plot maps the canvas into the printable rect and adds the margin border, so the SVG page equals the paper size exactly.
- **Why:** what is on screen is exactly what plots. Confirmed: page width/height attrs equal the paper dims (e.g. A3 portrait -> 297x420mm).

### Orientation included
- **Chose:** portrait/landscape select that swaps preset dimensions; disabled for custom (custom uses literal width/height).
- **Why:** paper sizes without orientation cannot frame landscape images; it is a one-line swap and integral to the feature.

### Fit the canvas to the container in both axes
- **Chose:** displayW = min(availW, availH / paperAspect) from the stage-wrap content box (computed padding subtracted); the renderer sets explicit canvas style width/height.
- **Why:** portrait sheets would otherwise overflow vertically. Driving size from the container (not the canvas's own clientWidth) avoids a width:100% feedback loop. Verified: A4 portrait 653x952, A3 landscape 1052x729, custom 400x150 -> 1052x360, all fit the 1400x1000 viewport.

## Architectural Notes
- printableWidthMm renamed to printableMm (axis-agnostic, used for width and height); still floors at 1mm so a margin >= half the page cannot divide by zero.
- PlotOptions gained heightMm; the page height is now explicit from the paper, not derived from the canvas.
- Renderer signature: render(params, page: PageGeometry, previewProfile|null). PlotPreview interface removed - the page geometry is shared by the normal and preview paths, and the preview only needs the profile (pxPerMm derived from the page).
- state.paperDimensions(settings) is the single source mapping paperSize+orientation+custom -> {widthMm, heightMm}; used by both render and export.
