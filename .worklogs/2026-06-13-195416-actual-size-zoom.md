# 100% (actual size) zoom toggle

**Date:** 2026-06-13 19:54
**Scope:** src/state.ts, src/render/canvas.ts, src/main.ts, src/ui/controls.ts, src/styles.css, tests/e2e/app.spec.ts

## Summary
Added a "100% (actual size)" checkbox in the Plotter section. At 100% the printable page area is displayed at real physical mm (browser 96px/inch reference) so letter sizes can be checked against a ruler before plotting. The plot itself is unchanged; only the on-screen presentation scales.

## Context & Problem
After the paper-size work the user wanted to verify real letter sizes (relevant given the 0.3 mm pen legibility floor). A "100%" view must show the SAME plot at actual size, not re-tile a different cell count.

## Decisions Made

### Scale CSS only; keep the layout basis = host width
- **Chose:** the layout/internal buffer is still derived from the host-fit displayW (unchanged), and only canvas.style width/height changes with zoom.
- **Why:** cols = floor(displayW/(0.62*cellPx)) depends on the host-derived displayW, not the canvas style size. So changing the style size cannot change the cell count - the plot stays identical, which is the whole point of a "check" view. Verified: internal buffer width is byte-identical across the toggle (653 px in both modes); only the displayed size changes (653 px -> 718 px = 190 mm).

### Set the style in mm, not a computed px constant
- **Chose:** canvas.style.width = `${printW}mm` / height = `${printH}mm` at 100%.
- **Why:** the browser owns the mm->px reference (96/inch); most accurate available without knowing the monitor, and the e2e can assert the exact '190mm' string. A4 printable -> 190mm x 277mm -> 718 x 1047 px.

### Show the printable area at actual size
- **Chose:** display the printable area (paper minus margins), since that is what the canvas already represents and where the letters live.
- **Why:** the letters' real size is what matters for legibility; the margin border is not rendered anyway.

### safe center for the stage wrapper
- **Chose:** align-items/justify-content: safe center on #stage-wrap.
- **Why:** at 100% (and large papers) the canvas overflows the container; plain centered flex clips the top-left and blocks scrolling. `safe` falls back to start when overflowing so the whole page is scrollable.

## Architectural Notes
- render() gained a 4th param zoom ('fit'|'actual'), default 'fit'; ZoomMode + zoom live in PlotSettings (alongside the other view toggle, preview).
- "Actual size" is approximate: real monitors are not exactly 96 CSS px/inch, so 1 CSS mm != 1 physical mm on every display. Best a web app can do without a calibration step.
- Minor: a scrollbar appearing at 100% could shave a few px off host width on the next render (tiny cols shift); negligible / absent with overlay scrollbars.
